## 3. Header variations

| Variation | Description | Page count | Pages |
|-----------|-------------|------------|--------|
| **Pattern A — Main header** | `<header class="header" id="header">` with `.mid-container`, `.header__left` (logo), `.header__middle` (verdicts), `.header__right` (phone). Sibling `<div class="header-nav-wrap" id="header-nav-wrap">` with `<button class="mobile-menu-toggle">` and `<nav class="header-nav-row" id="primary-nav">` (dropdowns). | **114** | Home, personal-injury, parking-lot-accident-lawyer-los-angeles, los-angeles-car-accident-lawyer, legal/*, motor-vehicle (hub), premises-liability (hub), and all other content pages except the two below. |
| **Pattern B — Sticky header** | `<header class="sticky-header">` with `.header-content`, `.header-col--logo`, `.header-col--proof`, `.header-col--call`. `<nav class="header-nav-wrap">` is the nav element (flat links only). **No** `mobile-menu-toggle`, **no** dropdowns, **no** tap-to-call bar. | **2** | `motor-vehicle/bus-accident-lawyer-los-angeles/`, `premises-liability/negligent-security-lawyer-los-angeles/` |

**Recommendation:** Replace Pattern B with Pattern A on the two silo pages so header and nav match the rest of the site. Do **not** change the header on `/`, `/personal-injury/`, `/parking-lot-accident-lawyer-los-angeles/`, or `/los-angeles-car-accident-lawyer/` (they already use Pattern A).

## 9. Recommended fixes (order of operations)

Do **not** heavily modify: `/`, `/personal-injury/`, `/parking-lot-accident-lawyer-los-angeles/`, `/los-angeles-car-accident-lawyer/`.

1. **Unify header and footer (2 pages)**  
   - **Pages:** `motor-vehicle/bus-accident-lawyer-los-angeles/`, `premises-liability/negligent-security-lawyer-los-angeles/`.  
   - **Action:** Replace sticky header with main header (Pattern A). Replace minimal footer with full footer.  
   - **Result:** Same header/footer/mobile nav as the rest of the site; mobile menu no longer broken on these two URLs.

2. **Canonical tags (4 pages)**  
   - **Pages:** `legal/accessibility/`, `legal/disclaimer/`, `legal/results-disclaimer/`, `legal/terms/`.  
   - **Action:** Add `<link rel="canonical" href="https://www.insiderlawyers.com/legal/.../">` to each.  
   - **Result:** Clear canonical URL per page; lower risk of duplicate or weak indexing.

3. **Breadcrumbs (107 pages)**  
   - **Pages:** All content pages that currently lack breadcrumbs (excluding `/` and `/thank-you/`).  
   - **Action:** Add breadcrumb block using existing pattern (e.g. `nav.breadcrumb`, aria-label, optional BreadcrumbList JSON-LD). On protected pages, add only the breadcrumb block; do not rewrite content or change layout.  
   - **Result:** Consistent wayfinding and stronger topical signals for SEO.

4. **Nav script consistency (optional)**  
   - **Action:** Ensure every page with the main header loads `site-nav.js`; remove duplicate inline nav/scroll logic where present (e.g. `what-to-do-after-car-accident-california/`).  
   - **Result:** Single behavior for mobile menu and header scroll; easier maintenance.

5. **Leave unchanged**  
   - Thank-you: keep as orphan and noindex.  
   - Protected pages: no heavy edits; only additive changes (e.g. breadcrumbs) where recommended above.
